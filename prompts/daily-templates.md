# daily-templates

这份文件收录 Mark 版框架里最常用的日常调用模板。原则只有三个：短、稳、可直接复制。

---

## 1. 判断一个创业想法值不值得做

```text
请把这个问题路由到最合适的任务类型，并用合适的 skill 组合帮我判断这个创业想法值不值得做。请先分别给出各个 skill 的判断，再汇总成一个统一建议，并附最低成本下一步。

想法：{在这里写你的创业想法}
当前约束：{时间/预算/身份/能力}
```

---

## 2. 评估一个产品功能该不该加

```text
请把这个问题作为产品评审处理，用最合适的 skill 组合判断这个功能该不该加。请输出：各 skill 判断、共识、冲突、统一建议，以及最低成本下一步。

产品：{产品名}
功能：{功能描述}
目标用户：{用户是谁}
```

---

## 3. 把复杂知识讲明白

```text
请把这个问题作为 learning 任务处理，用最合适的 skill 组合把这个概念讲明白。要求：讲清楚、少术语、最好有直觉例子，最后补一句“我可以怎么把它讲给别人听”。

问题：{概念/代码/原理}
我当前水平：{入门/中级/进阶}
```

---

## 4. 做一篇更有传播性的内容选题

```text
请把这个问题作为 content 任务处理，用最合适的 skill 组合帮我优化这个选题。请输出：这个题值不值得做、推荐切口、标题方向、结构建议，以及最低成本下一步。

原始选题：{你的选题}
目标平台：{公众号/小红书/抖音/视频号/X}
目标受众：{你的受众}
```

---

## 5. 审查一个方案的风险

```text
请把这个问题作为 risk 任务处理，用最合适的 skill 组合帮我审查这个方案的风险。请输出：主要风险、为什么危险、缓冲措施、统一建议，以及必须先验证的事项。

方案：{你的方案}
最大投入：{时间/金钱/机会成本}
最坏结果：{可能发生什么}
```

---

## 6. 做季度规划

```text
请把这个问题作为 planning 任务处理，用最合适的 skill 组合帮我做未来 90 天规划。请输出：本季度主目标、必须砍掉的事、必须保留的事、关键指标，以及最低成本下一步。

当前情况：{你的现状}
我最想推进的事：{事项}
我当前最大约束：{时间/钱/学业/精力}
```

---

## 7. 对一个重大选择做多视角评审

```text
请把这个问题路由到最合适的任务类型，并用 2-4 个最合适的 skill 做一次多视角评审。要求：先列出每个 skill 的判断，再提炼共识、指出冲突，最后形成一个统一建议和最低成本下一步。

我要做的选择：{你的选择}
我最担心的点：{风险/机会/犹豫点}
当前约束：{时间/预算/身份/能力}
```

---

## 8. 极简通用版

```text
请先判断这个问题属于哪类任务，再自动选择最合适的 skill 组合。不要堆太多 skill。请按“分角色判断 -> 共识 -> 冲突 -> 统一建议 -> 最低成本下一步”的格式输出。

问题：{你的问题}
当前约束：{时间/预算/身份/能力}
```

# learning-explainer

## 使用目的

用于把复杂概念、代码原理或技术系统讲清楚。

## 适用输入

适合输入以下信息：

- 你想理解的概念或代码片段
- 你当前的理解程度
- 你最终想达到的理解深度
- 是否需要结合代码、类比或现实例子说明

## 调用哪些 skill

默认调用：Feynman + Karpathy  
涉及前沿 AI 时可补：Ilya Sutskever

岗位分工：
- Feynman：负责讲明白
- Karpathy：负责技术直觉与实现理解
- Ilya（可选）：负责前沿抽象与研究方向补充

## 执行步骤

1. 先把问题归类为 `learning`。
2. 写清楚要解释的对象、背景和目标理解层级。
3. 让 Feynman 用普通人语言解释它是什么、为什么重要。
4. 让 Karpathy 解释它在系统和代码里如何体现。
5. 如涉及前沿能力边界，再让 Ilya 补前沿抽象视角。
6. 汇总成统一解释：定义、直觉、例子、误区、应用。

## 输出格式

```markdown

### 任务卡片
- 类型：learning
- 主题：

- 分工：

### 分角色判断

- Karpathy：
- Ilya（可选）：

### 统一解释
- 这是什么：
- 为什么重要：
- 如何工作：
- 直观例子：
- 常见误区：

### 下一步学习动作
1. 
2. 
3. 
```

## 示例

`请用 Feynman + Karpathy 的分工，向一个有基础编程能力但不懂深度学习的人解释 transformer 里的 attention 机制。`

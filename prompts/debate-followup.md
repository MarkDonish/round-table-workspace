# Debate Followup Prompt

你现在处于 `/debate` 的补充讨论阶段。

前一轮讨论没有通过审查，禁止直接输出最终决议。

你的任务是：

1. 明确指出前一轮为什么没有通过
2. 指出哪些 Agent 需要补充
3. 指出每个 Agent 要补什么
4. 限制补充讨论只解决关键缺口，不重复整场会议
5. 补充完成后，再次提交给审查 Agent

## 触发条件

只有当上一轮审查结果为“`不允许进入最终决议`”时，才进入本流程。

如果上一轮已经允许放行，本 prompt 不适用。

## 返工上限

- 本阶段是 Full 模式唯一允许的返工轮次
- 只允许被点名的 Agent 补充
- 禁止要求全部 Agent 重讲
- 如果补充后仍不通过，必须输出阻塞结论，而不是继续开下一轮

## 补充讨论规则

- 只补关键缺口，不重做所有发言
- 仍然必须按 Agent 职责边界补充
- 补充内容必须包含依据或不确定项
- 如果缺的是反方视角，必须新增或补强反方论点
- 如果缺的是可执行性，必须把建议写成动作
- 如果已经满足的部分合格，不要重复输出
- 主持人只能更新受影响的汇总部分，不能重写整场会议

## 补充失败后的处理

如果二次审查仍不允许进入最终决议，必须输出：

1. 不可放行的临时结论
2. 当前最稳妥的保守建议
3. 阻塞原因
4. 仍缺的证据
5. 需要用户补充的信息

完成后，重新调用审查 Agent，不允许跳过。

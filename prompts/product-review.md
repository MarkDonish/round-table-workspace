# product-review

## 使用目的

用于判断一个产品功能、需求或路线选择是否应该进入当前版本。

## 适用输入

适合输入以下信息：

- 产品名称与核心用户
- 当前阶段：MVP / 增长期 / 成熟期
- 待评审的功能是什么
- 希望功能带来的用户行为或业务指标变化
- 预估开发成本与上线风险

## 调用哪些 skill

默认调用：Steve Jobs + Zhang Yiming + Elon Musk  
高风险时可补：Taleb

岗位分工：
- Steve Jobs：产品核心价值与体验聚焦
- Zhang Yiming：反馈闭环、机制设计、迭代效率
- Elon Musk：第一性原理拆解复杂度
- Taleb（可选）：系统风险与不可逆影响审查

## 执行步骤

1. 先把问题归类为 `product`。
2. 明确功能要解决的用户问题、核心指标和开发代价。
3. 让 Steve Jobs 判断它是否服务于核心体验。
4. 让 Zhang Yiming 判断它是否能进入数据反馈与迭代系统。
5. 让 Elon Musk 判断当前方案是否存在冗余与伪复杂度。
6. 如功能引入高系统风险，再加入 Taleb 做风险制衡。
7. 汇总为“保留 / 砍掉 / 延后 / 降级实现”的统一建议。

## 示例

`请用 Steve Jobs、Zhang Yiming、Elon Musk 的岗位分工，评审“AI 自动生成知识图谱”功能是否应该进入当前版本，并给出统一结论。`

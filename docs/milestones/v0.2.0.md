# v0.2.0 Milestone

> Status: active milestone plan.
> Last updated: 2026-04-29.

## Goal

`v0.2.0` makes Round Table Workspace easier to enter, easier to validate, and
less tied to host-specific runtime folders. The release should strengthen the
local-first `/room`, `/debate`, and `/room -> /debate` protocols without
claiming live support that has not been validated.

## Out Of Scope

- Do not claim every third-party `host-live` lane is supported.
- Do not require `provider-live` support for the local-first mainline.
- Do not say a `fixture pass`, mock-provider pass, wrapper presence, or config
  preflight is live support.
- Do not move canonical protocol behavior into `.claude/skills/` or any single
  host adapter.
- Do not rewrite all runtime code at once; core extraction should remain
  incremental and test-backed.

## Claim Boundary

The milestone may claim checked-in protocol, schema, fixture, docs, CLI, and
local validation improvements. It must not claim universal host support or real
provider support until matching live validation evidence exists under the
release rules in `docs/release-candidate-scope.md`.

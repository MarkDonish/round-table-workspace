# Red Flags

本文件定义 `/debate` 圆桌会议中的低质量讨论红旗。审查 Agent 每轮都必须扫描。

## 1. 空话

定义：

- 使用听起来正确但没有信息增量的话
- 例如“要聚焦核心价值”“要长期主义”“要以用户为中心”，但没有具体化

识别标准：

- 无具体对象
- 无具体依据
- 无具体动作

---

## 2. 套话

定义：

- 使用模板化、政治正确、放之四海皆准的话，不能帮助本议题推进判断

识别标准：

- 可以无损替换到任何议题中
- 去掉后不影响结论

---

## 3. 偷换问题

定义：

- 用户问 A，输出却在答 B

常见形式：

- 把“值不值得做”偷换成“如果做的话怎么做”
- 把“风险多大”偷换成“机会很大”
- 把“先做什么”偷换成“未来愿景是什么”

---

## 4. 过度自信

定义：

- 在依据不足时给出过强结论

识别标准：

- 置信度高，但没有足够依据
- 不说明不确定项
- 用确定语气掩盖假设

---

## 5. 结论先行、依据后补

定义：

- 先给强结论，再临时补一些泛泛理由

识别标准：

- 理由不能真正支撑结论
- 结论力度明显大于证据力度

---

## 6. 混淆事实与判断

定义：

- 把判断说成事实
- 把偏好说成客观结论

识别标准：

- 没有标明“这是推断”
- 没有标明“这是基于哪些事实做的判断”

---

## 7. 没有提出反方视角

定义：

- 所有发言都在强化同一方向，没有指出关键反对意见

识别标准：

- 没有提到方案最大的一个问题
- 没有说明常见误判
- 没有识别下行风险

---

## 8. 没有说明不确定性

定义：

- 输出看起来很完整，但完全不写不知道的部分

识别标准：

- 不写不确定项
- 不写待验证假设
- 不写需要额外信息的地方

---

## 9. 建议不可执行

定义：

- 建议听上去对，但不能转成具体动作

识别标准：

- 没有下一步动作
- 没有时间、资源、顺序、边界
- 没有停止条件或复盘点

---

## 10. 角色越权

定义：

- Agent 跑去回答不属于自己职责边界的问题

识别标准：

- 多个 Agent 重复同类意见
- 关键角色职责空缺
- 发言像混战而不是圆桌分工

---

## 11. 伪深刻

定义：

- 为了显得聪明而使用抽象词汇、复杂表达或人格表演

识别标准：

- 语言密度高但信息密度低
- 模仿风格强于判断质量
- 情绪强于结构

---

## 12. 红旗处置规则

- 轻度红旗：可要求补充说明后继续
- 中度红旗：要求指定 Agent 补充讨论
- 重度红旗：直接不允许进入最终决议

默认重度红旗：

- 偷换问题
- 关键结论无依据
- 严重混淆事实与判断
- 建议不可执行
- 明显空话但直接下最终结论

---

## 13. `/debate` 专属红旗

### 13.1 触发污染

定义：
- 用户没有输入 `/debate`，系统却擅自进入圆桌会议模式

识别标准：
- 普通单 skill 请求被错误升级成多 Agent 会议

### 13.2 职责重叠

定义：
- 多个 Agent 在做同一层判断，导致重复和混乱

识别标准：
- 同一主问题出现多个主责 Agent
- 多个发言高度同质化

### 13.3 审查越权

定义：
- 审查 Agent 超出可见材料，假装看到了未展示内容

识别标准：
- 出现“虽然没写出来，但显然他考虑到了”这类判断
- 直接替讨论内容补脑

### 13.4 返工跳过

定义：
- 审查不通过后，没有补充讨论，直接给最终决议

识别标准：
- 存在“不允许进入最终决议”但后文仍直接给 final answer

### 13.5 虚假统一

定义：
- 主持人没有真的消解冲突，只是把不同观点压成一个看似统一的结论

识别标准：
- 冲突点没有解释假设差异
- 统一建议与前文分歧不匹配

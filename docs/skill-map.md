## Mark 版优先使用建议

### 1. 高优先级 skill

这些是 Mark 最常用、最稳妥的一组：

- Paul Graham：判断创业问题是否真实
- Munger：防止自欺和错误配置
- Taleb：控制风险和下行暴露
- Feynman：把复杂东西讲清楚
- Zhang Xuefeng：把抽象判断拉回学生身份和现实约束
- Jobs：做产品聚焦
- Zhang Yiming：设计反馈回路和内容-产品机制

### 2. 条件性高优先级 skill

这些在特定场景很强，但不宜泛用：

- Karpathy：当问题是 AI / 代码 / 深度学习理解时
- Naval：当问题是季度规划、长期方向、职业路径时
- MrBeast：当目标是内容冷启动、传播、选题优化时
- Musk：当问题是系统冗余、成本过高、方案太重时

### 3. 谨慎使用 skill

- Ilya：适合前沿判断，不适合日常执行主导
- Trump：只适合极少数强势表达场景，不适合作为默认决策视角

## Mark 版使用建议

### 1. 优先按岗位选，不按名人选

推荐的选法是：

- 需要机会判断 -> Paul Graham
- 需要产品取舍 -> Jobs
- 需要机制设计 -> Zhang Yiming
- 需要解释复杂概念 -> Feynman / Karpathy
- 需要风险审查 -> Taleb / Munger
- 需要现实校准 -> Zhang Xuefeng
- 需要传播优化 -> MrBeast
- 需要长期方向 -> Naval

### 2. 先看偏差，再决定是否纳入

如果某个 skill 的偏差在当前问题里会被放大，就不要让它主导最终结论。

例子：

- 做高风险投入判断时，不应让 MrBeast 或 Trump 成为主导视角。
- 做 AI 学习解释时，不应只让 Ilya 输出高抽象结论，必须配 Feynman 或 Karpathy。
- 做个人规划时，Naval 很适合给方向，但必须配 Munger 或 Taleb 做现实与风险校准。
- 做学生创业选择时，任何偏宏大叙事的 skill 都必须配 Zhang Xuefeng 或 Taleb。

### 3. 组合不是越多越好

- 2 个 skill：适合单一目标问题
- 3 个 skill：适合机会判断 + 落地 + 风险制衡的问题
- 4 个 skill：只用于跨多个维度的复杂任务

超过 4 个 skill，通常说明分工不清或问题定义不清。

### 4. Mark 版默认偏好

- 先做能验证的事
- 先做能积累的事
- 先做不会把自己搞崩的事
- 先做能产生下一步真实信号的事

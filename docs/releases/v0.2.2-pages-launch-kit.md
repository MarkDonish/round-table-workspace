# v0.2.2-pages-launch-kit Release Notes

This release turns Round Table Workspace from a repo-only project into a cleaner
public launch surface.

## What Changed

- Added `docs/index.html`, a static GitHub Pages landing page that explains the
  project in one screen and links back to the repository.
- Added `docs/index.html` so the repository can be published through GitHub
  Pages using the main branch /docs folder setting without requiring a workflow
  token scope.
- Added `./rtw launch-kit`, a structured checklist command for launch assets,
  topics, demo links, and public positioning.
- Updated `README.md` with the public demo URL and the current release label.
- Updated `docs/launch-copy.md` so external posts include both the GitHub repo
  and the live demo URL.
- Added `tests/test_launch_surface.py` to keep the public launch surface,
  launch-kit command, application packet, and release note wired together.

## Why This Matters

For a first-time visitor, the project now has three stronger entry points:

1. GitHub README for developers.
2. GitHub Pages landing page for screenshots, sharing, and quick comprehension.
3. `rtw launch-kit` for machine-readable launch assets and topic checklist.

## Claim Boundary

This release does not add new host-live or provider-live support. It improves
public packaging around the existing local-first, fixture-backed surfaces.

## Verification

Run:

```bash
python3 -m unittest discover -v
./rtw launch-kit --json
./rtw doctor --quick --quiet
./rtw release-check --include-fixtures --quiet
```

# v0.2.0-alpha Release Notes

Release date: 2026-04-29

Release type: alpha local-first development release

Support scope: Codex local mainline plus v0.2.0 protocol/core/validation
development scope

## Decision

`v0.2.0-alpha` packages the full v0.2.0 development-canvas implementation on
`main` for local-first validation and review.

This release may be used for:

- Unified `./rtw` CLI discovery, validation, doctor, evidence, release-check,
  interactive, and demo entrypoints.
- README and launch quickstarts for the local-first path.
- Checked-in transcripts and fixture/mock golden demo material.
- Protocol spec, room schema, debate schema, and `/room -> /debate` handoff
  schema.
- Decision-quality rubric and reviewer prompt alignment.
- Initial `roundtable_core/` abstractions for validation, state store, prompt
  rendering/parsing, agent registry, and host adapter contracts.
- Skill generation, generated skill summaries, slimmed skill references, and
  skill drift checks across Codex and Claude adapters.
- Decision-quality evals, regression fixtures, source-truth consistency checks,
  claim-boundary dashboard, and aggregate release-check automation.
- Documentation index, milestone, roadmap, and release governance updates.

## What Changed Since `v0.1.3`

- Added the remaining v0.2.0 protocol and runtime contracts after the earlier
  alpha-entry work.
- Added host-neutral `roundtable_core` modules for prompt rendering/parsing,
  state recording, agent lens registry, and fixture-backed host adapter tests.
- Added checked-in handoff schema/fixture support so `/room -> /debate` can be
  validated as an explicit contract.
- Added fixture/mock decision-quality evals and regression fixtures for
  room/debate/handoff flows.
- Added `scripts/generate_skills.py`, `scripts/check_skill_drift.py`,
  `scripts/check_source_truth_consistency.py`,
  `scripts/claim_boundary_dashboard.py`, `scripts/run_regression_fixtures.py`,
  and `scripts/release_check.py`.
- Added `./rtw interactive` and `./rtw demo startup-idea` as claim-safe
  fixture/mock user experience paths.
- Updated docs to mark all 27 v0.2.0 development-canvas task IDs as implemented
  on `main`.

## What This Release Does Not Claim

This alpha release must not be described as a universal all-host production
release.

The following lanes are intentionally outside the supported claim unless their
own live validation reports are generated:

- Claude Code host-live support on any machine/account that has not passed
  `claude_code_live_validation.py`.
- OpenCode host-live support before a host matrix row reports
  `matrix_status=live_passed`.
- Gemini CLI, Aider, Goose, or Cursor Agent host-live execution before a host
  matrix row reports `live_passed`.
- Real Chat Completions-compatible provider-live execution before valid
  `.env.room` and `.env.debate` files exist and the live wrapper passes.
- Universal stability across every possible local agent host.
- A fully wired natural-language `./rtw room` or `./rtw debate` host runtime;
  those CLI commands remain safe stubs until the host runtime is connected.

## Verification Commands

The release should be checked with:

```bash
python3 -m unittest discover -v

./rtw release-check \
  --include-fixtures \
  --strict-git-clean \
  --state-root /tmp/round-table-release-check-v0.2.0-alpha

./rtw doctor \
  --quick \
  --state-root /tmp/round-table-doctor-v0.2.0-alpha

./rtw demo startup-idea \
  --state-root /tmp/round-table-demo-v0.2.0-alpha

python3 .codex/skills/room-skill/runtime/release_candidate_report.py \
  --include-fixture-runs \
  --strict-git-clean \
  --state-root /tmp/round-table-release-candidate-v0.2.0-alpha \
  --output-json /tmp/round-table-release-candidate-v0.2.0-alpha.json \
  --output-markdown /tmp/round-table-release-candidate-v0.2.0-alpha.md
```

After tagging, audit the published release path:

```bash
python3 .codex/skills/room-skill/runtime/post_release_consumer_audit.py \
  --ref v0.2.0-alpha \
  --state-root /tmp/round-table-post-release-consumer-audit-v0.2.0-alpha
```

Pass criteria:

- Unit tests exit `0`.
- `./rtw release-check --include-fixtures --strict-git-clean` exits `0`.
- `./rtw doctor --quick` exits `0` and returns `overall_status=pass`.
- `post_release_consumer_audit.py --ref v0.2.0-alpha` exits `0` after the tag
  exists.
- `p0_blockers` is empty.
- `provider_url_required` remains false for the Codex local mainline.

# v0.1.0 Release Notes

Release date: 2026-04-26

Release type: stable local-first release

Support scope: Codex local mainline plus validated Mac Claude Code host-live evidence

## Decision

`v0.1.0` is the first stable local-first release for the repository's current
claim-safe support scope.

It promotes the validated `v0.1.0-rc4` scope without widening the claim beyond
the evidence currently checked in.

This release may be used for:

- Codex local mainline `/room`.
- Codex local mainline `/debate`.
- Codex local mainline `/room -> /debate`.
- Checked-in protocol, prompts, skills, runtime bridges, and validation
  harnesses.
- Root `LAUNCH.md` clone-friendly startup path.
- Agent consumer self-check for new clones and local agent hosts.
- Claude Code project-skill discovery structure as an adapter layer.
- Default Claude Code CLI host-live support on the validated Mac account where
  `claude_code_live_validation.py` returned
  `claimable_as_default_claude_code_host_live=true`.
- Generic local agent adapter contract with fixture-backed validation.
- Third-party local agent JSON wrapper and validation matrix tooling.
- Repo-local development checkpoints as historical continuity evidence.
- Source-truth boundary audit tooling.
- Chat Completions-compatible fallback/mock regression tooling.

## What Changed Since `v0.1.0-rc4`

- Promoted the checked and tagged rc4 support scope to stable `v0.1.0`.
- Updated launch/readiness/changelog references from rc4 to the final release
  note.
- Kept all live-claim boundaries unchanged.

## What This Release Does Not Claim

This release must not be described as a universal all-host production release.

The following lanes are intentionally outside the supported claim unless their
own live validation reports are generated:

- Claude Code host-live support on any machine/account that has not passed
  `claude_code_live_validation.py`.
- Gemini CLI, OpenCode, Aider, Goose, or Cursor Agent host-live execution before
  a host matrix row reports `live_passed`.
- Real Chat Completions-compatible provider-live execution before valid
  `.env.room` and `.env.debate` files exist and the live wrapper passes.
- Universal stability across every possible local agent host.

## Verification Commands

The release should be checked with:

```bash
python3 .codex/skills/room-skill/runtime/agent_consumer_self_check.py \
  --state-root /tmp/round-table-agent-consumer-self-check-v0.1.0

python3 .codex/skills/room-skill/runtime/release_readiness_check.py \
  --include-fixture-runs \
  --strict-git-clean \
  --state-root /tmp/round-table-release-readiness-v0.1.0 \
  --output-json /tmp/round-table-release-readiness-v0.1.0.json

python3 .codex/skills/room-skill/runtime/release_candidate_report.py \
  --include-fixture-runs \
  --strict-git-clean \
  --state-root /tmp/round-table-release-candidate-v0.1.0 \
  --output-json /tmp/round-table-release-candidate-v0.1.0.json \
  --output-markdown /tmp/round-table-release-candidate-v0.1.0.md
```

Pass criteria:

- `agent_consumer_self_check.py` exits `0` and returns `ok=true`.
- `release_readiness_check.py` exits `0`.
- `release_candidate_report.py` exits `0`.
- `p0_blockers` is empty.
- `release_decision` is `ready_for_codex_local_mainline_scope`.
- `ready_to_claim` includes `default Claude Code host-live support on the
  validated Mac account`.
- `real_host_live_passed` includes `claude_code`.
- `provider_live_ready` remains false unless a real provider-live run passes.
